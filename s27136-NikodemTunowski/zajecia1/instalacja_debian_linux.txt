wyjaśnienie systemow plikow:
ext4 journaling file system - Ext2 ale z journalingiem danych, lepiej 
zoptymalizowany niz ext3 przez co jest szybszy. Journaling to tak jakby 
historia plików, pozwala ona na odzyskanie plików w przypadku ich utraty, 
uszkodzenia lub przypadkowego nadpisania.
ext3 journaling file system - ext2 z journalingiem danych, jest wolniejszy niz 
ext2 ze wzgledu na dodatek w postaci journalingu.
ext2 - podstawowy system plików, nie zawiera journalingu.
btrfs journaling file system - plik przed zapisem znajduje sie w innym 
miejscu, dopiero po ostatnim zapisie oryginalny plik jest podmieniony.
JFS journaling file system - Używany głównie na serwerach. Max wielkość pliku 
- 4 PiB
XFS jounraling file system - Następca JFS, o wiele większa wielkość pliku - 8 
exbibajtów minus 1 bajt
FAT16 file system - 16 bitowy system plików do małych partycji, limit partycji 
i plików to tylko 2GB.
FAT32 File System - Nowsza i lepsza wersja FAT16, limit partycji to 16GB a 
pliku - 4GB
swap area - System plikow uzywany do partycji swap.
physical volume for encryption - partycja uzywana do szyfrowania plikow.
physical volume for RAID - partycje do konfiguracji RAID.
physical volume for LVM - takie partycje nie zmieniaja etykiety po restartach 
w danym clusterze.
do not use this partition - partycja nieuzywana, przydatne jesli chce sie 
zrobic z partycja cos niestandardowego lub wgrac inny system plikow
wyjaśnienie mount options:
discard - wydaje polecenia TRIM dla danych które nie są uzywane, opcja dobra 
dla dysków SSD, ponieważ zwalnia regularnie miejsce na dysku które co 
przyśpiesza zapisy i wydłuża żywotność dysku, jednak może ona spowodować 
utrate danych gdy nie jest uzywana prawidlowo.
noatime - wyłącza aktualizacje atrybutu atime w plikach, który przechowuje 
czas ostatniego dostępu do pliku, włączenie tej opcji ogranicza liczbe 
operacji zapisu na dysku, co zwiększa szybkość systemu.
nodiratime - to samo to noatime tylko że dla katalogów
relatime - sprawia że system uzywa czasu wzglednego do aktualizacji atrybutu 
atime dla plikow i folderow, czyli ten atrybut aktualizuje sie tylko wtedy gdy 
czas dostepu do pliku/folderu jest pozniejszy niz czas jego ostatniej 
modyfikacji, pozwala to zwiększyc zywotnosc dysku ze wzgledu na mniejsza ilosc 
operacji.
nodev - włączenie tego sprawia że system plików nie uznaje plików 
odpowiedzialnych za urządzenia fizyczne, np dyski, czy porty szeregowe, 
zwiększa to bezpieczeństwo jednak niektóre aplikacje mogą wymagać dostępu do 
tych plików co spowoduje problemy.
noseuid - wyłączenie interpretowania bitów SUID i SGID, odpowiadaja one za 
działanie z uprawnieniami użytkownika lub grupy, do której należą, a nie z 
uprawnieniami użytkownika, który je uruchamia, bez nich komendy su, sudo i 
passwd nie będą działały, jednak brak tej interpretacji może miedzy innymi 
chronić poufne dane przed dostępem przez nieautoryzowanych użytkowników
noexec - zabrania wykonywania jakichkolwiek plików wykonywalnych/bibliotek na 
danej partycji, zwieksza to bezpiecznestwo danych, ale sprawia ze zadne 
programy/skrypty na danej partycji nie beda dzialaly.
ro - partycja tylko do odczytu, blokuje zapis/nadpis danych.
sync - wymuszenie synchronizacji danych z pamieci podrecznej systemu plikow do 
dysku przed zamontowaniem systemu plikow. gwarantuje integralnosc danych i 
bezpieczenstwo.
usrquota - limit zajmowanego miejsca partycji przez danego uzytkownika, 
zapewnia sprawiedliwy podzial dysku i zapobiega temu ze jakis uzytkownik 
zajmie miejsce innym.
grpquota - to samo co usrquota tylko ze limity są stosowane dla grup kont.
user_xattr - Rzoszerzone atrybuty użytkownika, czyli dodatkowe metadane które 
są dołączane do plików i folderów, poszerza funkcjonalność systemu plików, 
pozwala na kompresje plików.
Reserved blocks - Miejsce na dysku niedostępne dla zwyklego uzytkownika, 
zapobiega przypadkowemu nadpisaniu lub usunięciu ważnych danych znajdujących 
się na początku partycji.
Typical usage - pozwala wybrać sposób uzywania partycji, w zaleznosci co 
wybiore zmienia sie sposob indeksowania plikow. do wiekszych plikow lepiej 
zastosowac rzadsze inody.
Bootable flag - z tej partycji system będzie sie bootowal, nalezy wybrac 
partycje gdzie znajduje sie folder /boot, inaczejk system sie nie uruchomi.
Package manager - pomijam, ponieważ nie mam dodatkowych mediow instalacyjnych. 
mozna tego uzyc jesli ma sie dodatkowe pakiety na innym nosniku.

wirtualki. Przydaje się to do administracji systemem, transferu plików i wielu 
innych.

zainstalowany inny bootloader to nie ma potrzeby instalacji gruba.
